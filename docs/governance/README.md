# EGC Governance

Documents that define how subsystems are classified, how skills and
agents are placed and adapted, and how capability surfaces are
selected. Read `SUBSYSTEM-MAP.md` first.

| File | Scope |
|---|---|
| `SUBSYSTEM-MAP.md` | Formal classification of every top-level subsystem (ACTIVE / GENERATED / ARCHIVAL / DORMANT / LEGACY / DEPRECATED) and the governance policy per class. |
| `SKILL-PLACEMENT-POLICY.md` | Rules for placing a new skill under the correct namespace in `skills/` |
| `skill-adaptation-policy.md` | When and how to adapt or fork an existing skill |
| `capability-surface-selection.md` | Criteria for surfacing a skill into a harness install profile |

For runtime contracts (command/agent map, session adapter), see
`../runtime/`. For onboarding contributor process, see the root
`CONTRIBUTING.md` and `../README.md`.
